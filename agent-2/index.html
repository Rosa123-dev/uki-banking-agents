What You're Building

What Your Process Mapper Can Do

This agent takes the pain out of process documentation. Feed it almost anything — and it will produce a structured, formatted process map ready to drop into your tool of choice.

Audio & Transcript Input

Paste a meeting transcript or upload an audio recording. The agent extracts process steps automatically — no manual note-taking needed.

PowerPoint & Document Input

Hand it a deck or a Word doc describing a process and it will convert the content into a structured flow — no reformatting required.

Web Scraping

Give it a URL or describe a standard industry process. It will research and build a baseline process map for you to refine.

Mermaid Code Output

Get clean, ready-to-render Mermaid diagram code — works in Microsoft Copilot, VS Code, Confluence, Notion and more.

Visio-Ready Table

Outputs a structured table (Step, Actor, Action, Decision, Next Step) that maps directly into Visio's import format — one click to diagram.

BPMN Formatting

Apply BPMN notation standards, swim lanes, gateway logic and event types — ideal for formal process documentation.

The Most Important Part

Writing Your Agent's Instructions

Your instructions define what format the agent outputs, what inputs it accepts, and how it structures its diagrams. The more specific you are, the more consistent and useful the output will be every time.

✅ What a strong prompt includes

  • Your preferred default output format (Mermaid / Visio / BPMN)
  • The notation style — e.g. BPMN 2.0, flowchart, swim lanes
  • Preferred colour coding or shape conventions if any
  • What inputs it should accept (audio transcript, doc, URL, free text)
  • How to handle ambiguity — ask clarifying questions or make assumptions
  • Whether to include a written summary alongside the diagram
  • Level of detail — high-level overview vs granular step-by-step
  • Actor/role labelling preferences (e.g. use job titles not names)

❌ Common mistakes to avoid

  • Not specifying a format — it'll pick one at random each time
  • Forgetting to mention swim lanes if you want them
  • Not saying how detailed to go — you'll get inconsistent depth
  • Skipping colour/style rules — output looks different every time
  • Not telling it how to handle unclear inputs
  • Forgetting to ask for a text summary — diagrams without context are hard to review
  • Not specifying actor labels — it may use names instead of roles
💡 Example Prompt — Copy, Edit & Make It Yours
Full System Prompt Example

You are an expert process mapping assistant supporting analysts and consultants at Accenture. Your job is to take any input I provide and convert it into a clear, structured process map. INPUTS YOU SHOULD ACCEPT: - Pasted text descriptions of a process - Meeting transcripts or audio transcripts (I will paste the text) - Content from PowerPoint slides or Word documents (I will paste the text) - A URL — scrape the page and infer the process from its content - Free-form verbal descriptions ("here's roughly how it works...") DEFAULT OUTPUT FORMAT: [CHOOSE ONE: Mermaid code / Visio-ready table / BPMN] Unless I ask for a different format, always use the above. If I ask you to switch format, remember that preference for the rest of the conversation. FORMATTING RULES: - Always use swim lanes to separate actors/roles - Label nodes with action verbs (e.g. "Review Document", not "Document Review") - Use diamonds for decisions, rectangles for tasks, rounded rectangles for start/end - Colour code by actor: [define your colours here, e.g. Analyst = blue, Manager = purple] - Include a numbered written summary below every diagram listing each step in plain English NOTATION STANDARD: [CHOOSE: BPMN 2.0 / Standard flowchart / Agile-style] LEVEL OF DETAIL: Default to mid-level (5–12 steps). If I say "high level" give me 3–6 steps. If I say "detailed", go granular with sub-steps. HANDLING AMBIGUITY: If the input is unclear or incomplete, ask me up to 3 clarifying questions before generating the map — don't guess at critical decision points. ACTORS: Always refer to people by their role/job title, never by name. Always end your response by asking: "Would you like me to adjust the format, level of detail, or add any missing steps?"

🚀 Ways to Take It Further
01

Build a template library

Ask the agent to generate baseline maps for common banking processes (KYC, loan approval, complaints handling) and save them as reusable starting points.

02

Layer in risk & controls

Instruct it to annotate each step with relevant risk flags or control checkpoints — turning a process map into a risk assessment tool.

03

Compare as-is vs to-be

Generate two versions of the same process — current state and future state — and ask it to summarise the key differences and efficiency gains.
