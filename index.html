SCHEDULED PROMPTS

Keep the Momentum Going

Scheduled Prompts let you run a Copilot prompt automatically on a schedule — turning Copilot from a reactive tool into a proactive assistant.

What are Scheduled Prompts?

Instead of typing the same prompt every morning, you write it once, run it once, and tell Copilot to repeat it on a schedule. Copilot runs the prompt for you and posts the result back into Copilot chat — with an optional email notification. Think of it as setting a recurring alarm, but instead of waking you up it briefs you.

Available in
microsoft365.com/chat Copilot in Teams Copilot in Outlook

⚠️ Requires a Microsoft 365 Copilot licence. Up to 10 scheduled prompts at a time.

How to set one up

  1. 1
    Write and run your prompt once — you can only schedule it after it's been run at least once.
  2. 2
    Hover over the prompt you just ran and select "Schedule this prompt".
  3. 3
    Choose your schedule — start date, frequency, number of runs, and whether you want an email notification.
  4. 4
    Hit Save. Copilot will run it automatically from now on.

To manage your scheduled prompts: open the ⋯ menu in Copilot and select "Scheduled prompts". From there you can edit, pause, run immediately, or delete.

Good examples to try

🌅 Start of Day

"Summarize today's meetings, agendas, and key documents."

Set this to run each weekday morning so you start the day already briefed — no manual inbox check needed.

📬 Inbox Triage

"Summarize emails that still need my attention and flag anything urgent."

Run this daily at lunchtime to catch anything that slipped through your morning review.

✅ Mid-Week Follow-ups

"Remind me what messages and tasks from earlier this week still need action."

Schedule for Wednesday afternoon — catch anything that got buried in the week's noise.

📝 End of Day Wrap-up

"Summarize action items and decisions from meetings I attended today."

Set for 5pm daily. You'll finish every day with a clean list of what needs to happen next.

Wondering How We Built This?

The website you're looking at right now is itself a demonstration of what's possible when a human with the right ideas works alongside AI. Here's how it came together.

Designed by a Human

The session concept, the four agent ideas, the learning journey, the prompt frameworks, and the decision to make it hands-on rather than a slide deck — all of that came from a person. AI can build, but it needs someone to know what's worth building.

Built with Claude

Every page of this website — the landing page, all four agent guides, the colour scheme, the sample data files, and the interactive elements — was written in code by Claude, Anthropic's AI. No web developers. No designers. Just a conversation.

Hosted on GitHub Pages

When it came to publishing the site, Claude walked through every step — setting up a GitHub repository, structuring the file folders, enabling GitHub Pages, and troubleshooting along the way. No prior technical knowledge required.

This is the point of today's session. The technology is accessible. The question is whether you have the curiosity to pick it up — and the ideas to point it at something that matters.
